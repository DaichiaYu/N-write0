# N-write0

N-write0 是一個為長期 AI 協作設計的個人想法與寫作工作台。

它協助使用者把零碎觀察、草稿、研究資料與逐步形成的判斷，整理成可查找、可修訂、可追溯的內容。AI Agent 可以依倉庫中的入口、索引與狀態標記讀取相關資料，而不必把所有筆記視為同等成熟或同等可信。

這個公開版本提供一套可直接使用的基礎結構，不包含任何特定使用者的文章、立場、產品資料或研究內容。

## 可以用來做什麼

- 收納尚未成形的觀察、問題與靈感；
- 發展長文、文章、研究筆記或其他正式內容；
- 管理草稿、修訂稿與已發佈版本；
- 整理反覆出現的概念、系列與相關資料；
- 讓 AI 查找既有內容，並注意內容目前的狀態與限制；
- 在想法改變時保留版本脈絡，而不是直接覆蓋舊內容。

N-write0 以寫作作為主要整理方式，但不限定使用者一定要成為作者。它也可以作為研究、產品思考、個人判斷與長期 AI 協作的資料工作台。

## 入口

- `AGENTS.md`：AI Agent 第一入口與基本協作規則
- `writing/目錄.md`：文件導航
- `writing/article-index.md`：正式內容索引
- `writing/01-inbox/writing-dashboard.md`：近期工作區
- `writing/01-inbox/topic-candidates.md`：待發展主題
- `scripts/check-repo-integrity.py`：只讀結構檢查

## 主要結構

```text
writing/
  00-workflow/        流程、模板、詞彙與整理規則
  01-inbox/           尚未成熟的想法與工作筆記
  02-drafts/          已形成方向的草稿
  03-revision/        修訂中或準備分享的版本
  04-published/       已定稿或已公開版本
  05-series/          系列索引
  06-concepts/        概念地圖與概念字典
  07-research-notes/  外部資料與來源筆記
  08-ai-consensus/    可供後續 AI 協作參考的整理層
```

## 最快開始方式

1. 先讀 `AGENTS.md` 與 `writing/目錄.md`。
2. 把第一則零碎想法放進 `writing/01-inbox/YYYY/MM/YYYY-MM-DD-inbox.md`。
3. 需要發展、查找或整理內容時，讓 AI 依入口文件與現有索引工作。
4. 定期執行 `python scripts/check-repo-integrity.py` 檢查結構。

## 使用提醒

- Inbox 內容仍可能只是片段、假設或待確認問題。
- 外部資料、AI 延伸與使用者自己的判斷應保持區分。
- 重要內容更新時，應同步檢查相關索引與版本紀錄。
- AI 可以協助整理與提出建議，但不應替使用者自行確認立場。

N-write0 提供的是可以開始工作的骨架；內容、判斷與使用方式，會隨使用者逐步形成。
