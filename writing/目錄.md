# N-write0｜文件目錄

本文件提供倉庫導覽。AI 應先閱讀 `AGENTS.md`，再依任務讀取必要文件；不要因為看到目錄就一次讀取或重構整個倉庫。

## 主要入口

| 路徑 | 用途 |
|---|---|
| `README.md` | 人類使用說明與倉庫概覽 |
| `AGENTS.md` | AI 第一入口、任務限制與禁止事項 |
| `writing/目錄.md` | 本導覽文件 |
| `scripts/check-repo-integrity.py` | 只讀結構檢查 |
| `writing/article-index.md` | 正式內容索引 |

| `writing/00-workflow/tag-vocabulary.md` | 既有標籤詞彙 |

## 內容資料夾

| 路徑 | 用途 |

## 常用檔名

```text
YYYY-MM-DD-topic-slug.zh-TW.md
YYYY-MM-DD-topic-slug.en.md
YYYY-MM-DD-topic-slug.idea.md
YYYY-MM-DD-topic-slug.map.md
YYYY-MM-DD-inbox.md
```

完整規則以 `writing/00-workflow/ai-naming-rules.md` 為準。已有名稱時優先沿用，不要自行建立第二套命名方式。

## 使用限制

- 只讀取完成目前任務所需的文件；
- 建立內容前先檢查同資料夾、相關索引與既有名稱；
- 不要自行新增 workflow、資料夾層級、索引種類或狀態；
- 不要因內容相似就自動合併文章、概念或系列；
- 不要把 Inbox、外部資料、AI 摘要或候選內容當成使用者正式立場；
- 修改後只同步直接受影響的連結與索引；
- 發現結構缺口或語意衝突時，列出問題並等待使用者決定，不要自行補完整系統。
