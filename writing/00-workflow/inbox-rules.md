# Inbox Rules

## 成熟度分層

| 類型 | 用途 | 路徑 | 進 article index |
|---|---|---|---|
| Raw Capture | 幾句想法、未形成主張 | `01-inbox/YYYY/MM/YYYY-MM-DD-inbox.md` | 否 |
| Structured Idea Note | 已有核心直覺與問題 | `01-inbox/YYYY/MM/YYYY-MM-DD-topic.idea.md` | 否 |
| Idea Map | 一組母題或可拆分方向 | `01-inbox/YYYY/MM/YYYY-MM-DD-topic.map.md` | 否 |
| Topic Candidate | 值得發展但未成稿 | `01-inbox/topic-candidates.md` | 否 |
| Formal Draft | 已有 core claim 與正文 | `02-drafts/YYYY/MM/` | 是 |

## Raw Capture 最小格式

```markdown
## IDEA-YYYY-MM-DD-001｜暫定名稱

### Raw Thought
### Possible Tags
### Possible Concepts
### Fuzzy Terms
### Possible Use
### Open Questions
### Status
raw
```

## Structured Idea Note 必備內容

- 目前狀態
- 核心直覺
- 可能角度
- 使用者目前立場
- Not My Claim
- Open Questions
- 可能用途

## 升級規則

- Raw Capture 不自動進 topic candidates。
- Idea Map 先建立一個母 candidate，不把每個分支都變成任務。
- 有清楚 core claim 與正文骨架後，才升級 Formal Draft。
- 任何 inbox 內容都不得直接升級成 AI Consensus。
