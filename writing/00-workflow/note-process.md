# 筆記流程

N-write0 的流程不是從「寫完整文章」開始，而是從保留想法開始。

## 1. Capture｜先丟進來

把原始想法放進：

```text
writing/01-inbox/
```

可以很粗糙，只要保留：

- 我看到什麼？
- 我為什麼覺得這件事值得記？
- 我的直覺是什麼？
- 我還不確定什麼？

## 2. Sort｜整理成方向

當一個想法開始反覆出現，可以判斷它比較像：

- 個人觀察
- 產品想法
- 文章主題
- 研究筆記
- 概念定義
- 專案決策
- 其他

## 3. Draft｜整理成草稿

初步整理後放進：

```text
writing/02-drafts/YYYY/
```

這個階段不用追求完整，先讓想法可以被自己或 AI 讀懂。

## 4. Revision｜修訂與準備分享

如果內容準備公開、投稿、發給別人、放進簡報或整理成正式文件，放進：

```text
writing/03-revision/YYYY/
```

## 5. Publish｜已公開或已定稿

已公開、已投稿、已交付或已定稿的內容，放進：

```text
writing/04-published/YYYY/
```

## 6. Concept｜整理反覆出現的核心概念

如果某個詞、判斷或方法反覆出現，可以整理到：

```text
writing/06-concepts/
```

## 7. AI Consensus｜給 AI 的共識層

如果某些判斷會影響長期協作，可以整理到：

```text
writing/08-ai-consensus/
```

例如：

- 我已經確定的立場
- 我暫時不採用的方向
- AI 常誤解我的地方
- 需要反覆提醒的限制
