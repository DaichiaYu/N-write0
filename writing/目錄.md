## 任務路線

### 1. 記錄零碎想法

```text
Raw Capture          → writing/01-inbox/YYYY/MM/YYYY-MM-DD-inbox.md
Structured Idea Note → writing/01-inbox/YYYY/MM/YYYY-MM-DD-topic.idea.md
Idea Map              → writing/01-inbox/YYYY/MM/YYYY-MM-DD-topic.map.md
```

### 2. 建立正式草稿

```text
writing/02-drafts/YYYY/MM/YYYY-MM-DD-topic.zh-TW.md
```

同步 article index、月份 manifest、concept vocabulary、MOC 與 weekly digest。

### 3. 找舊想法

依序查 dashboard、topic candidates、concept vocabulary、article index、MOC、manifest、weekly / monthly，最後才全文搜尋。

### 4. 週整理 / 月整理

先跑 integrity checker。週整理負責增量整理；月整理才提出較穩定的共識更新建議。

### 5. 更新共識

先進 `consensus-candidates.md`，經使用者確認後，才移入 user stance、decision logic、not-my-claim 或 open questions 索引。

## 命名規則

```text
YYYY-MM-DD-topic.zh-TW.md
YYYY-MM-DD-topic.en.md
YYYY-MM-DD-topic.idea.md
YYYY-MM-DD-topic.map.md
YYYY-MM-DD-inbox.md
```

## 維護原則

- 新增 workflow 或模板時，同步更新本目錄與 `AGENTS.md`。
- 新增 MOC 時，更新 concept vocabulary。
- 新增系列時，使用 `writing/05-series/<series-slug>/README.md`。
- 內容形成長期判斷時，只提醒是否更新 AI consensus，不自動升級。
